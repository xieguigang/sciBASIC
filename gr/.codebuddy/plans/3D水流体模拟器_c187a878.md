## 用户需求概述

在现有 `physics\Particles`（2D 流体 SPH 引擎）基础上，新增对 3D 空间中水的模拟能力，且不破坏现有 2D 代码；随后新建一个 WinForm 桌面应用，借助 `Microsoft.VisualBasic.Imaging\Drawing3D` 的 3D 渲染 API 与升级后的流体引擎，构建一个可交互的 3D 水模拟器。

## 核心功能

- **3D 流体引擎升级**：在 physics 项目中新增自包含的 3D SPH 水体模拟（双精度 3D 向量、3D 空间哈希邻域搜索、3D 归一化核函数、3D 粒子与 3D 流体主循环），支持长方体容器内边界碰撞与重力。
- **参数可视化调节**：WinForm 内通过 PropertyGrid 控件实时设置模拟参数（粒子数、平滑半径、重力、时间步长、目标密度、压力系数、黏度、黏滞阻尼、容器尺寸等）。
- **长方体边界线框**：用线条绘制容纳水模拟的 3D 长方体空间边界（12 条棱），随相机旋转/透视投影同步显示。
- **鼠标旋转观察**：通过鼠标拖拽旋转相机欧拉角（AngleX/AngleY）改变观察视角，滚轮缩放视距。
- **晃动窗口扰动**：拖拽/晃动窗口时在桌面位移产生的速度/加速度，映射为容器的惯性扰动力作用于所有水粒子，观察晃动后水的流动与回落。
- **实时渲染**：每个模拟步将粒子投影为带速度/深度着色的水点，构成可见的 3D 水体。

## 技术栈选择

- 流体引擎：沿用 physics 项目（VB.NET, net10.0，命名空间 `Microsoft.VisualBasic.Imaging.Physics`），保持自包含，不引入对 Imaging 的循环引用（physics 当前未引用 imaging）。
- 3D 渲染：复用 `Microsoft.VisualBasic.Imaging.Drawing3D`（`Point3D`、`Camera`、`Surface`、`PainterAlgorithm`）。
- 桌面应用：新建 WinForm 项目（VB.NET, `net10.0-windows`, `UseWindowsForms=true`, WinExe），参考 `ModelViewer` 项目的窗体/面板/相机交互模式。
- 渲染桥接：WinForm 层将引擎输出的 `Vector3` 桥接为 `Drawing3D.Point3D` 进行旋转与投影。

## 实现方案

### 总体策略

在 `physics\Particles` 中按"镜像 2D 引擎"的方式新增一套 3D 实现：新建 `Vector3`（仿 `Vector2.vb`）、`FluidKernels3D`（采用 3D SPH 归一化常数）、`Particle3D`、`Grid3D`（3D 空间哈希，27 邻域）与 `FluidEngine3D`（SPH 主循环）。引擎对外暴露 `Entity`（粒子集合）、容器 `BoxSize` 与一组可绑定到 PropertyGrid 的参数属性；并新增 `DisturbAccel` 外部扰动加速度属性（由窗口晃动注入，引擎内按衰减系数回落）。WinForm 端建立定时器驱动 `RunSimulationStep`，并用 `Camera` 将长方体边界与水粒子投影为 GDI+ 线条/圆点。

### 关键技术决策

- **自带 `Vector3` 而非引用 Imaging**：physics 项目未引用 imaging，为避免循环依赖并保持引擎可独立编译/单测，在 physics 内新建与 `Vector2` 同构的 `Vector3`（x/y/z 双精度，运算符 + - * /，Zero/One，Random(box)，Magnitude，Cross/Dot 可选）。
- **3D 核函数归一化**：3D SPH 与 2D 归一化常数不同（如 Poly6=315/(64πh⁹)、SpikyPow3=15/(πh⁶)、其导数 45/(πh⁶)、SpikyPow2=15/(πh⁵)），新建 `FluidKernels3D` 复用 `FluidKernels` 的方法签名但替换常数，保证物理正确。
- **3D 邻域搜索**：`GridDynamics` 仅支持 2D 网格（9 邻域）。新增 `Grid3D` 模块，采用字典哈希（cx,cy,cz）→ 粒子数组，提供 27 邻域 `SpatialLookup3D`，将邻域查询复杂度由 O(n²) 降为 O(n·k)，与 2D 性能一致。
- **窗口晃动扰动**：Form 上挂一个高频 `Timer`（约 60Hz）采样 `this.Location`，与上一次采样求位移 Δ、速度 v、加速度 a；容器加速度反向（惯性力 = -a）乘以尺寸比例缩放后写入 `engine.DisturbAccel`；引擎每步将扰动叠加到重力项并对 `DisturbAccel` 做指数衰减，使水自然回落。比仅监听 `LocationChanged` 更平滑可靠（拖拽时系统事件可能丢帧）。
- **边界与渲染**：长方体 8 角点减质心居中后，经相机旋转+投影用 `g.DrawLine` 绘制 12 条棱（灰色/青色）；水粒子同样居中、并行旋转+投影，按 z 降序以蓝色实心圆点绘制（深度/速度着色），复用 `ModelViewer.SceneRenderer.ToScreen` 的"Rotate+Project"管线。

### 性能与可靠性

- 引擎主循环沿用 `Parallel.For`（与 2D 一致）做密度/压力/黏度/位置更新的数据并行；邻域网格重建每步一次，O(n)。
- 渲染端采用双缓冲 `Panel`（仿 `RenderPanel`），投影并行算点坐标、串行绘制（GDI+ 不可并行）。
- `DisturbAccel` 衰减与速度截断避免数值爆炸；边界用 padding 防止粒子穿透。
- 保持 2D `FluidEngine`/`FluidKernels`/`Particle` 完全不变，仅在 `Particles` 新增文件，降低回归风险。

## 实现注意事项

- physics 为 SDK 风格工程，默认 glob 包含 `Particles\*.vb`，新文件无需改 vbproj 的 `Compile` 列表（注意避免落入 `physics_Test/**` 与 `test/**` 排除项，新文件放在 `Particles\` 下即可）。
- 渲染时务必将世界坐标减质心居中（`center = BoxSize * 0.5`），否则相机绕原点旋转会偏离模型中心。
- PropertyGrid 绑定的参数对象应用 `[Category]`/`[Description]` 标注，修改后通过"应用"按钮或属性 setter 触发引擎重建/参数同步；粒子数变更需重建引擎。
- `Camera.Screen` 每帧从 `canvas.ClientSize` 同步；窗口缩放时相机视距可通过 `FitView` 思路自适应。
- 复用现有 `Camera` 的 `Rotate(points)`（SIMD 批量）与 `Project(pt)` 透视投影，保证与库内其它渲染一致。

## 设计风格

采用深色工程控制台风格（Dark Glassmorphism + 科技蓝），突出 3D 水体与线框边界。整体布局：顶部菜单/工具条（开始/暂停、重置视角、应用参数），左侧固定面板放置 PropertyGrid（参数调节），主区域为 Dock=Fill 的双缓冲渲染面板，底部状态栏显示 FPS、粒子数、相机角度与晃动加速度。

### 各区块设计

- **顶部工具条**：深色半透明背景，按钮为"开始/暂停模拟"、"重置视角"、"应用参数"、"显示边界"开关，悬停高亮。
- **左侧参数面板**：`PropertyGrid` 绑定 `FluidParameters`，分类（模拟/物理/容器/扰动），深色主题；支持数值微调。
- **主渲染面板**：黑色/深蓝渐变背景，绘制青灰色长方体 12 条棱（线框），水粒子以蓝色实心圆点呈现，按速度大小由浅蓝到深蓝着色，近大远小带透视，拖拽平滑旋转。
- **状态栏**：显示实时 FPS、粒子数、相机 AngleX/Y、视距、当前晃动加速度模长。

### 交互

- 鼠标左键拖拽：旋转相机（改 AngleY=dx*0.4, AngleX=dy*0.4）；滚轮：缩放视距；右键拖拽：平移。
- 拖拽晃动窗口：状态栏加速度数值实时跳动，水体产生对应惯性扰动并随后回落。
- 所有控件双缓冲、无闪烁；参数修改后点击"应用参数"重建/同步引擎。
